# Sprint Results

This file tracks sprint outcomes and observations. All quantitative data lives in `sprints.json` -- this file is for narrative context that numbers can't capture.

## How to Use This File

After each sprint, add an entry below with:
- What you built (phase/milestone reference)
- What surprised you
- Whether gates passed on first attempt (and what they caught if not)
- Any skill or process changes you're considering

The import pipeline (`tools/import_sprint.py`) handles all quantitative data. This file is for the story behind the numbers.
